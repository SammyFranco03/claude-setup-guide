Designer's Field Guide: From Vibe Code to Live AI Prototype

Get Claude running in your terminal, connect it to Figma, build an AI-powered POC, and deploy it to a real cloud environment — no engineering background required.

Setup time: ~45–60 min · Questions? Message Samantha on Teams or Slack, or email samantha.franco@cvshealth.com

Prerequisites

Please complete these before the setup session — some take time to process:

  • A Mac (this guide is written for macOS)
  • Python 3 downloaded and installed — the installer needs it to run (download from python.org)
  • Admin permissions on your laptop to install software (request admin access here — this can take 1–2 days, so do it now)
  • Connected to the CVS Health network or VPN (Zscaler or Cisco AnyConnect)
Don't skip the admin request! Without admin permissions, the installer will fail. Request admin access here now so it's approved by the session. If you're not sure whether you have admin access, try installing any app — if it asks for a password you don't have, you need to request it.

Setup Guide

1

Install CVS Code

~5 minutes to download

This is our company's installer — it downloads everything you need in one go. Make sure you're on the CVS network or VPN, then copy and paste this command into Terminal and hit Enter:

Terminal
curl -fsSL https://cvscode.prod.cvshealth.com/download/cvscode/install.sh | bash
Important: Use your machine's Terminal app, not VS Code's built-in terminal or any embedded shell. The installer may not work correctly in those.
2

Update Your PATH

~2 minutes

After the installer finishes, you need to tell your terminal where to find the new cvscode command. This is like adding a new app to your dock so you can find it.

3a. Run this command:

Terminal
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc

3b. Press ⌘ Cmd + Q to quit Terminal completely, then reopen it with ⌘ Cmd + Space and type "Terminal".

3c. Verify the install:

Terminal
cvscode --version

You should see a version number. If you see "command not found", close and reopen Terminal again.

3

Run the Environment Setup

~15–30 minutes (grab a coffee!)

Now we'll let CVS Code set up everything else automatically. It will install several tools behind the scenes — you just need to start it and tell it what to do.

Terminal
cvscode

Once inside, type this prompt to kick off the automated setup:

Inside CVS Code
Set up my development environment
This takes 15–30 minutes. The setup process will install several tools. Retry cycles are normal — let it run and don't close the terminal.
4

Sync Skills & Launch Claude Code

~2 minutes

Almost there! After the environment setup completes, run these two commands to download Claude's skills and launch it:

Inside CVS Code
/skill-sync

Then launch Claude Code:

Inside CVS Code
launch claude

Claude Code's interface should now appear. You're in!

Next time you want to use Claude Code, just open Terminal, type claude, and press Enter.
5

Add the Figma MCP Server

~1 minute

This is the magic step — it lets Claude see and interact with your Figma files. Run this single command in your Terminal:

Terminal
claude mcp add --transport http figma https://mcp.figma.com/mcp

That's it. One line and Figma is connected.

6

Authenticate with Figma

~1 minute

The first time you use a Figma command, Claude will pop open your web browser and ask you to log into Figma. Just sign in like normal and click "Allow" to authorize the connection. This only happens once.

To test it, paste a real Figma URL from one of your files and ask Claude about it.

7

Enable the Figma Plugin

~1 minute

One last thing — enable the Figma plugin to unlock all the design superpowers. While Claude is running, type:

Inside Claude
/plugins

Look for figma in the list and enable it. You're all set!

How to Start Claude Again Later

Once everything is set up, open Terminal any time and run this command to start working with Claude:

Terminal
cd ~/Desktop/MCP && claude

That's it — two commands chained together. The first navigates to your MCP folder, the second launches Claude with Figma connected.

Why Build a Live AI Prototype

Figma handles static screens. Voiceflow handles agent logic. Neither shows how UI changes in real time based on LLM output. A live prototype fills that gap.

Real LLM Interactions

See how the agent responds probabilistically — not scripted click-throughs, but actual AI-generated UI in the moment.

Real User Testing

Share a live link with users inside or outside CVS Health, not a Figma prototype limited to pre-set flows.

UX Ownership of Prompting

Establish a design perspective on how AI agents should behave — before engineering builds it their way.

Faster Engineering Handoff

Hand off working code instead of specs. CAP deployment can reduce infrastructure setup from months to minutes.

Real example: Joe Lowmiller on the Health 100 team vibe-coded a full health agent prototype (Heyo) — complete with rich UI components, a modular prompting system, and fake patient data — then deployed it to CAP for internal user testing. All built without a dedicated engineering sprint.

Generate Fake Data

Ask Claude to create realistic test data as JSON files — no backend required. You can iterate to add edge cases.

Example prompt

3

Connect to Claude via Hyperion Gateway

Hyperion Gateway is the approved internal path to Claude at CVS Health. It generates an API key automatically — no manual credential setup required.

  • Ask CVS Code: "What AI backends do we have available for a web app I want to deploy?" — it will guide you to Hyperion Gateway.
  • Or go to User Settings → settings.json inside CVS Code — the Hyperion URL and key are listed there.
Watch out: If Claude tells you to manually create Anthropic, Gemini, or Artifactory credentials — stop. Use Hyperion Gateway instead. You cannot manually push to Artifactory; the CI pipeline handles that automatically.

What You Need to Do Before Deploying

Here's a checklist of the actual steps required before your first CAP deployment — some take a day or more, so start early.

All source code at CVS Health lives in GitHub Enterprise. Request access through AccessNow if you don't already have it. This is the foundation everything else depends on.

Visit cap-ui.prod.platform.cvshealth.com — you already have access. Familiarize yourself with the dashboard. Start with your personal team to create a test app and run Quick Start. No approvals needed for the dev environment.

If you want your whole product team (designers, product, engineers) to access and deploy your app, create a shared team in CAP. You can assign owners, members, and viewers. Members and owners can deploy; viewers can see the status.

In CAP, create an Application then run Quick Start. It scaffolds your CI/CD pipeline, links your GitHub repo, and sets up everything needed to deploy containers — automatically. You don't need to configure Kubernetes, Artifactory, or Docker yourself.

Dev deployments are frictionless. Production deployments require additional steps. See the full production getting-started docs for details. Key requirements:

  • Approved Change (CNG) in ServiceNow required for every production deployment
  • External ID aligned with a CVS Health Business Application ID (APM)
  • Assignment Group — create one in ServiceNow with your team members; all members need SN_PROD_ITIL entitlement (request via AccessNow)
  • Change approval — allow 1–7 business days; escalate if no response after 1 day
For POCs and user testing, stick to the dev environment — it has none of these requirements. Only move to production when engineering is ready to productionize the app.

Production deploy steps (once approved):

  1. Click "Run a Deployment" on your CAP service home page
  2. Select your deployment tag and add a memo (include the CNG number)
  3. Review configuration changes in the preview modal
  4. Select "Deploy Candidate" and monitor the pipeline
  5. Verify success through testing
  6. Close the change in ServiceNow

Iterating After Deploy

  1. Make changes in VS Code with Claude
  2. Tell Claude: "Commit and push my changes to main"
  3. CAP picks up the new commit automatically via the CI pipeline

Definitions

Plain-language explanations for every tool mentioned in this guide:

Terminal

A text-based app built into every Mac. Instead of clicking buttons, you type commands. Think of it as a chat window where you talk to your computer.

Claude Code

An AI assistant that runs inside your Terminal. You type questions in plain English and Claude responds — it can read files, write code, analyze designs, and more. Like chatting with ChatGPT, but right in your terminal with access to your files.

Figma MCP

A bridge that lets Claude see your Figma files. Without it, Claude has no idea what's in your designs. With it, you paste a Figma link and Claude reads the layout, colors, components, and can even generate code from them. MCP stands for "Model Context Protocol" — but you don't need to remember that.

Homebrew

An app store for your Terminal. Instead of downloading an installer from a website, you type one command and Homebrew downloads and installs the software for you.

Python

A programming language. You won't be writing Python — the CVS Code installer just needs it to run behind the scenes. Think of it like a required app that powers the installer.

Node.js

Another behind-the-scenes tool. Claude Code is built with Node.js, so it needs to be installed for Claude to run. The CVS Code setup installs this for you automatically.

CVS Code (cvscode)

Our company's custom installer and wrapper. It handles all the complicated setup so you don't have to do it manually. Think of it as the "setup wizard" for getting Claude running at CVS Health.

PATH

A list of folders where your computer looks for commands. When you type claude in Terminal, your computer checks the PATH to figure out where that program lives. We add one folder to this list during setup.

VPN

Virtual Private Network — the Zscaler or Cisco app you use to connect to the CVS network when working remotely. The installer needs it to reach CVS servers.

CAP — Cloud Application Platform

CVS Health's internal platform for deploying apps to the cloud. Think of it like a managed hosting service — you give it your code and it runs your app in a real cloud environment, generates a shareable URL, and handles all the infrastructure behind the scenes. Available to every CVS Health employee from day one, no tickets required for the dev environment. Open CAP →

Artifactory / JFrog

A storage system for packaged, security-scanned app code. When you deploy via CAP, your app gets automatically packaged into a "container image" and stored in Artifactory. You never interact with it directly — the CI/CD pipeline does this for you. If Claude tells you to manually set up a JFrog account or push something to Artifactory, stop — that's not how it works here.

Container / Containerized App

A self-contained package of your app and everything it needs to run — like a shipping container for software. CAP deploys containers. You don't need to build one yourself; CAP's Quick Start sets it up automatically when you connect your GitHub repo. Think of it as boxing up your app so it runs the same way everywhere.

CI/CD Pipeline

Continuous Integration / Continuous Deployment — an automated process that takes your code from GitHub, runs security checks, packages it, and deploys it to CAP whenever you push a new commit. You set it up once with CAP's Quick Start, and after that every push to GitHub triggers it automatically. You don't run it manually.

Hyperion Gateway

CVS Health's internal API gateway for connecting to Claude (Anthropic's AI). Instead of creating your own API key with Anthropic directly, you route requests through Hyperion Gateway — it handles authentication and generates a key for you. This is the approved, secure way to power AI features inside your CVS apps. Find the URL and key in User Settings → settings.json inside CVS Code.

GitHub Enterprise

CVS Health's internal version of GitHub — a platform for storing and versioning source code. All code that gets deployed through CAP must live in GitHub Enterprise. If you're vibe coding with Claude, the generated code needs to be committed and pushed here before CAP can deploy it. Different from github.com — this is an internal, company-managed instance.

Apigee

CVS Health's API gateway for exposing apps to users outside the CVS network (e.g., patients, customers). If you deploy to CAP and only need internal users (colleagues on VPN), you don't need Apigee. If you want external users to access your app for user testing, you'll need to work with the CAP team to set up Apigee and ensure no PHI is exposed.

Kubernetes

The infrastructure technology that CAP uses behind the scenes to run your containers in the cloud. You never interact with Kubernetes directly — the CAP team manages it for you. It's mentioned sometimes in error messages or documentation, but as a designer or non-engineer you can ignore it entirely.

ServiceNow

CVS Health's IT service management platform — used for submitting requests, tracking changes, and getting approvals. For CAP dev deployments you don't need it at all. For production deployments, you'll need to create a Change (CNG) in ServiceNow for every release and have it approved before deploying. Also used to request the SN_PROD_ITIL entitlement for team members.

AccessNow

CVS Health's system for requesting access to tools and entitlements — like GitHub Enterprise, production permissions, or the SN_PROD_ITIL entitlement required for production CAP deployments. If you hit a "permission denied" error on any CVS tool, AccessNow is usually where you go to request it.

GCP / Vertex AI

Google Cloud Platform is one of the cloud providers that CAP deploys to (currently GCP; AWS and Azure coming later in 2026). Vertex AI is Google's platform for running AI models — including Gemini. CAP is adding direct Vertex AI access in Q2 2026, which will let you connect your POC to Gemini and other GCP-hosted models without a separate approval process.

Troubleshooting

Your PATH wasn't updated. Run this, then close and reopen Terminal:

Terminal
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc

Claude Code may not have installed correctly. Once you have Homebrew installed, you can install the missing components — Claude Code needs NPM (Node.js) installed first:

1. Install Node.js (includes NPM):

Terminal
brew install node

2. Install Claude Code via NPM:

Terminal
npm install -g @anthropic-ai/claude-code

Then close and reopen Terminal.

Don't have Homebrew? Install it first by pasting this into Terminal:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Make sure you're on the CVS network or VPN. Try:

  1. Check your VPN is connected (Zscaler or Cisco)
  2. Close and reopen Terminal, then re-run the install command
  3. Retry cycles are normal — the installer may retry automatically

Homebrew is the Mac app store for Terminal — it helps install things like Python, NPM, gcloud, and more. Once you have Homebrew installed, there's a simple brew install command for each tool you need:

Terminal
brew install node
brew install python
brew install --cask google-cloud-sdk

Run whichever ones you need. If you don't have Homebrew yet, install it first:

Terminal
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Then close and reopen Terminal before running the brew install commands above.

  2. Restart Claude (/exit then claude)
  3. Type /mcp to check the server status

The CVS AIP brew formulas for npm and gcloud have incomplete installs — their lib/ directories are missing.

npm (nodejs formula)

The cvs/tools/nodejs formula only ships the node binary without node_modules/npm/. Workaround: use bun as your package manager instead — claude-code and codex are already installed via bun.

gcloud (gcloud formula)

The cvs/tools/gcloud formula only ships the shell wrapper without lib/gcloud.py and the rest of the SDK. Workaround: if you have Homebrew installed, you can install gcloud this way:

Terminal
brew install --cask google-cloud-sdk

Alternatively, install via the self-service app store:

Mac: Self-Service App Store (ColleagueZone)
