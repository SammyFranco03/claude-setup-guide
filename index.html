Setup Guide

Claude Code + Figma MCP

Get Claude running in your terminal so it can read your Figma files, generate code from your designs, and create diagrams — no coding experience required.

Total setup time: ~45–60 minutes • Stuck? Email samantha.franco@cvshealth.com

What Is All This Stuff?

Before we dive in, here's a plain-language explanation of every tool mentioned in this guide:

Terminal

A text-based app built into every Mac. Instead of clicking buttons, you type commands. Think of it as a chat window where you talk to your computer. You'll type commands here and see results as text.

Claude Code

An AI assistant that runs inside your Terminal. You type questions or requests in plain English, and Claude responds — it can read files, write code, analyze designs, and more. Think of it like chatting with ChatGPT, but right in your terminal with access to your local files.

Figma MCP

MCP stands for "Model Context Protocol" — but you don't need to remember that. It's basically a bridge that lets Claude see your Figma files. Without it, Claude has no idea what's in your designs. With it, you can paste a Figma link and Claude will read the layout, colors, components, and even generate code from them.

Homebrew

An app store for your Terminal. Instead of downloading an installer from a website, you type one command and Homebrew downloads and installs the software for you. We use it to install Python and other tools.

Python

A programming language. You won't be writing Python — the CVS Code installer just needs it to run behind the scenes. Think of it like a required app that powers the installer.

Node.js

Another behind-the-scenes tool. Claude Code is built with Node.js, so it needs to be installed for Claude to run. The CVS Code setup installs this for you automatically.

CVS Code (cvscode)

Our company's custom installer and wrapper. It handles all the complicated setup (installing Node.js, configuring connections, etc.) so you don't have to do it manually. Think of it as the "setup wizard" for getting Claude running at CVS Health.

PATH

A list of folders where your computer looks for commands. When you type claude in Terminal, your computer checks the PATH to figure out where that program lives. If it's not in the PATH, you get "command not found." We add one folder to this list during setup.

VPN

Virtual Private Network — this is the Zscaler or Cisco app you use to connect to the CVS network when working remotely. The installer needs to reach CVS servers, so make sure it's connected.

Before You Start

Please complete these before the setup session — some of these take time to process:

  • A Mac (this guide is written for macOS)
  • Python 3 downloaded and installed — the installer needs it to run (see Step 1)
  • Admin permissions on your laptop to install software (request through your admin portal — this can take 1–2 days, so do it now)
  • Connected to the CVS Health network or VPN (Zscaler or Cisco AnyConnect)
Don't skip the admin request! Without admin permissions, the installer will fail. Submit your request now so it's approved by the session. If you're not sure whether you have admin access, try installing any app — if it asks for a password you don't have, you need to request it.

1

Install Python 3

~5 minutes

Terminal is a text-based app on your Mac where you type commands instead of clicking buttons. Think of it like a chat window, but for talking to your computer.

Open it now: press ⌘ Cmd + Space to open Spotlight, type "Terminal", and hit Enter. A window with a blinking cursor will appear — that's your terminal.

First, let's check if Python is already on your machine:

Terminal
python3 --version

If you see a version number like Python 3.x.x, you already have it — skip to Step 2!

If you see an error or nothing happens, you need to install it. The easiest way is with Homebrew (a tool that helps you install things via Terminal):

Terminal
brew install python3
Don't have Homebrew either? Install it first by pasting this into Terminal:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Then close and reopen Terminal before running the brew command above.
2

Install CVS Code

~5 minutes to download

This is our company's installer — it downloads everything you need in one go. Make sure you're on the CVS network or VPN, then copy and paste this command into Terminal and hit Enter:

Terminal
curl -fsSL https://cvscode.prod.cvshealth.com/download/cvscode/install.sh | bash
Important: Use your machine's Terminal app, not VS Code's built-in terminal or any embedded shell. The installer may not work correctly in those.
3

Update Your PATH

~2 minutes

After the installer finishes, you need to tell your terminal where to find the new cvscode command. This is like adding a new app to your dock so you can find it.

3a. Add to your shell config

Run this command to add the path to your shell profile:

Terminal
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc

3b. Close ALL Terminal windows

Press ⌘ Cmd + Q to quit Terminal completely, then reopen it with ⌘ Cmd + Space and type "Terminal".

3c. Verify the install

Terminal
cvscode --version

You should see a version number. If you see "command not found", close and reopen Terminal again.

4

Run the Environment Setup

~15–30 minutes (grab a coffee!)

Now we'll let CVS Code set up everything else automatically. It will install several tools behind the scenes — you just need to start it and tell it what to do.

Terminal
cvscode

Once inside, type this prompt to kick off the automated setup:

Inside CVS Code
Set up my development environment
This takes 15–30 minutes. The setup process will install several tools. Retry cycles are normal — let it run and don't close the terminal.
5

Sync Skills & Launch Claude Code

~2 minutes

Almost there! After the environment setup completes, run these two commands to download Claude's skills and launch it:

Inside CVS Code
/skill-sync

Then launch Claude Code:

Inside CVS Code
launch claude

Claude Code's interface should now appear. You're in!

Next time you want to use Claude Code, just open Terminal, type claude, and press Enter.
6

Add the Figma MCP Server

~1 minute

This is the magic step — it lets Claude see and interact with your Figma files. Pick whichever method you prefer:

While Claude Code is running, paste this single command:

Inside Claude
/mcp add figma --transport http --url https://mcp.figma.com/mcp

That's it. One line and Figma is connected.

Open your settings file in Terminal:

Terminal
open -a TextEdit ~/.claude/settings.json

Find or add a "projects" section and include the Figma MCP server. Add this block to your existing settings:

Add to settings.json
"projects": {
  "/Users/YOUR_USERNAME": {
    "mcpServers": {
      "figma": {
        "type": "http",
        "url": "https://mcp.figma.com/mcp"
      }
    }
  }
}
Important: Replace /Users/YOUR_USERNAME with your actual home directory. Find it by running echo $HOME in Terminal.
7

Authenticate with Figma

~1 minute

The first time you use a Figma command, Claude will pop open your web browser and ask you to log into Figma. Just sign in like normal and click "Allow" to authorize the connection. This only happens once.

To test it, make sure Claude is running and try asking something like:

Inside Claude
Can you get the design context from this Figma link?
https://figma.com/design/YOUR_FILE_KEY/Your-File?node-id=123-456

Paste a real Figma URL from one of your files. Claude will prompt you to authenticate if you haven't already.

8

Enable the Figma Plugin

~1 minute

One last thing — enable the Figma plugin to unlock all the design superpowers. While Claude is running, type:

Inside Claude
/plugins

Look for figma in the list and enable it. This unlocks:

Read Designs

Claude can see and describe your Figma designs, including layout, colors, and typography.

Generate Code

Turn Figma components into production-ready React, HTML, or any framework code.

FigJam Diagrams

Create flowcharts, sequence diagrams, and other diagrams directly in FigJam.

Code Connect

Map Figma components to your codebase for seamless design-to-code workflows.

Troubleshooting

▶ "cvscode: command not found"

Your PATH wasn't updated. Run this, then close and reopen Terminal:

Terminal
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
▶ "claude: command not found"

Claude Code may not have installed correctly during the environment setup. Try running:

Terminal
npm install -g @anthropic-ai/claude-code

Then close and reopen Terminal.

▶ Installer hangs or fails

Make sure you're on the CVS network or VPN. The installer needs to reach cvscode.prod.cvshealth.com. Try:

  1. Check your VPN is connected (Zscaler or Cisco)
  2. Close and reopen Terminal, then re-run the install command
  3. Retry cycles are normal — the installer may retry automatically
▶ Figma MCP not connecting
  1. Make sure you've added the MCP server config correctly
  2. Restart Claude (/exit then claude)
  3. Type /mcp to check the server status
▶ Can't authenticate with Figma

Make sure your default browser is logged into Figma. The auth flow opens a browser tab — if you're not logged in there, it won't work.

▶ "Permission denied"

You need admin permissions on your machine. Request them through your company admin portal, then try again.
